# 🔍 로그인 무한 루프 및 프론트엔드 HMR 끊김 현상 핵심 원인 분석

> **분석 요약**: 쿠키와 CORS는 완벽하게 뚫렸지만, Axios 자체 설정 버그와 이전 컨텍스트의 잘못된 미들웨어 파일명(proxy.ts)으로 인해, 브라우저가 토큰 갱신 무한 루프에 빠지며 사이트 전체가 사실상 다운되는 현상을 발견했습니다.

## 1. 이슈 심층 분석

### 🚨 1. Axios 헤더 무한 루프 
네트워크 탭이 미친듯이 새로고침되는 진짜 이유는 **페이지 전체 로딩 루프가 아니라 API 재시도 무한 루프**였습니다.
1. `GET /api/insights` 요청 시 `accessToken`이 담기지 않아 **401 에러** 발생.
2. `api.ts` 응답 인터셉터가 이를 감지하고 `/api/auth/refresh` 요청 시도.
3. Path가 맞춰진 쿠키 덕분에 이 **Refresh 요청은 성공(200 OK)**하여 새 토큰을 받음.
4. 그러나 다시 원래(Original) 요청을 보낼 때, **Axios 1.x 버전의 엄격한 `AxiosHeaders` 객체 특성 때문에 `original.headers.Authorization = ...` 문법이 무시됨**.
5. 결과적으로 또 토큰 없이 `/insights`를 찌르고, 또 401을 받고, 또 성공적으로 Refresh하고... 를 초당 수십 번 반복하여 네트워크 자원이 고갈되며 Next.js의 웹소켓(HMR) 연결까지 끊어져 버림.

### 🚨 2. 잘려나간 Next.js 보안 미들웨어 (`proxy.ts` 네이밍 에러)
이전 세션에서 Next.js 16 규격이라며 `middleware.ts`를 `proxy.ts`로 변경한 조치가 있었습니다. **이는 치명적인 오해입니다.** Next.js는 `proxy.ts`라는 미들웨어 파일을 아예 인식하지 않습니다.
그 결과, 라우트 가드가 뚫려버려 비로그인 상태에서도 대시보드가 열렸으나 API만 401을 내뱉는 악순환이 완성되었습니다.

## 2. 해결 방안 (실행 계획)

### 단계 1: `api.ts` Axios 1.x 헤더 주입 방식 수정
무한 루프를 멈추기 위해, 최신 Axios 규격인 `headers.set()` 명시적 주입 방식으로 인터셉터를 전면 수정합니다.
- 변경점: `config.headers.Authorization = ...` ➡️ `config.headers.set('Authorization', ...)` 활용 및 `window.location.href` 강제 리로드 제거.

### 단계 2: 잃어버린 Next.js 미들웨어 복원
파일명 오류로 작동하지 않던 `src/proxy.ts`를 표준 규격인 `src/middleware.ts`로 **다시 변경**(Rename)하여 인증 없는 접속을 완벽하게 통제합니다. 
(이렇게 되면 무한 루프를 타기 전에 Next.js 단에서 먼저 깔끔하게 리다이렉트되어 시스템이 마비되지 않습니다.)
